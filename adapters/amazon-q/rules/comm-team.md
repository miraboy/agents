# Communication & Growth Team Rules

Apply these conventions to all content, copy, and communication deliverables in this project.

---

## Core Principle

Write for the **reader**, not for the brand.
Every word must be defensible. Consistency beats originality.
Tone must be replicable by any team member without ambiguity.

---

## Copywriting Standards

### Required frameworks (choose by context)

| Framework | Best for |
|-----------|----------|
| AIDA (Attention → Interest → Desire → Action) | Landing pages, display ads |
| PAS (Problem → Agitation → Solution) | Cold emails, problem-aware audiences |
| PASTOR (Problem → Amplification → Story → Transformation → Offer → Response) | Long-form sales pages |
| FAB (Features → Advantages → Benefits) | Product pages, feature launches |
| Before/After/Bridge | Brand awareness, transformation stories |

### Rules
- Headlines must promise a benefit or trigger curiosity — no generic titles
- First paragraph must hook in 3 seconds — lead with tension or benefit
- One idea per paragraph maximum
- Active voice; no jargon unless audience is confirmed expert
- Always deliver **2–3 headline variants** for A/B testing

---

## Landing Page Structure

```
HERO
  H1 (≤10 words): [Promise or transformation]
  Subheading: [Clarification + main benefit]
  CTA: [Action verb + benefit]
  Social proof: stat or testimonial

PROBLEM SECTION — name the pain, make reader feel seen

SOLUTION SECTION — brand as guide, not hero

BENEFITS (3–5) — benefit headline + 1-sentence explanation

SOCIAL PROOF — testimonials, logos, numbers

OFFER — what's included, pricing, guarantee

FINAL CTA — same action, reinforced with urgency or reassurance
```

---

## SEO Standards

Apply to all content targeting organic search:

- One H1 per page — matches primary keyword naturally
- Meta description: 150–160 characters, includes keyword, ends with soft CTA
- Verify search intent before writing: informational / transactional / navigational / commercial
- Internal linking: at minimum 2 links to existing relevant pages
- No duplicate content without canonical tag
- No black-hat techniques (keyword stuffing, hidden text, cloaking)
- Target Core Web Vitals: LCP < 2.5s, CLS < 0.1

**SEO Brief format:**
```
PRIMARY KEYWORD: [keyword] — Volume: [/mo] — Difficulty: [0–100]
INTENT: [informational | transactional | navigational]
OUTLINE: H1 → H2s → H3s
SECONDARY KEYWORDS: [list]
TARGET LENGTH: [word count]
INTERNAL LINKS TO: [pages]
META DESCRIPTION: [draft ≤160 chars]
```

---

## Brand Voice

Positional statement format:
> "For [target], [brand] is the only [category] that [unique benefit]."

Default tone (override with project brand guide if available):
- **Clear** over clever — simple words, short sentences
- **Confident** not arrogant — state facts, avoid superlatives
- **Human** not corporate — contractions acceptable, no buzzwords

Define per project:
- Words to always use
- Words to never use

---

## Brand Story (StoryBrand Framework)

```
Hero:      The customer (never the brand)
Problem:   External pain + internal frustration
Guide:     The brand — empathetic + credible
Plan:      3 clear steps
CTA:       Direct ("Start now") + Transitional ("See how it works")
Stakes:    What happens if they don't act
Success:   The transformation after using the product
```

---

## Email Marketing

- Subject: 30–50 characters, personalised if possible, no spam trigger words
- Preview text: complements (never repeats) the subject line
- One CTA per email — clear, singular action
- Plain-text version required alongside HTML
- Mandatory: unsubscribe link in footer
- A/B test order: subject line → CTA → layout

---

## Output Quality Standard

Every content deliverable must include:
1. Complete text — no placeholders or `[insert here]` gaps
2. 2–3 headline variants for A/B testing
3. Meta description (for SEO content)
4. CTA text — at least 2 formulations
5. Tone note — one sentence explaining the voice register and why
